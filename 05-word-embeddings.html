Word Embeddings

Short, dense, learned word vectors. Word2Vec's skip-gram with negative sampling (SGNS) trains a classifier to tell real neighbours from random words — and keeps the weights as embeddings. This is the most-weighted topic, so the SGNS update is worth mastering.

📘

Theory

Sparse counts → dense embeddings

Instead of long sparse TF-IDF vectors, learn dense vectors (length 50–1000). They have fewer parameters, generalise better, and capture synonymy (separate dimensions for car and automobile can't see their similarity; dense vectors can).

Word2Vec — predict, don't count

Big idea: don't count how often \(w\) occurs near "apricot"; instead train a classifier on a binary task — "is \(w\) likely to appear near apricot?" We don't care about the task; we keep the learned weights as embeddings. This is self-supervision: a word that really occurs near the target is the "correct answer", no human labels needed. We use skip-gram with negative sampling (SGNS).

The skip-gram classifier

Take a target word \(w\) and a context word \(c\). Their similarity is the dot product; squash it through the sigmoid to get a probability:

\[ P(+\mid w,c)=\sigma(\mathbf c\cdot\mathbf w)=\frac{1}{1+e^{-\mathbf c\cdot\mathbf w}} \] \[ P(-\mid w,c)=1-P(+\mid w,c)=\sigma(-\mathbf c\cdot\mathbf w) \]

For each positive pair we sample \(k\) negative words (\(k\approx2\text{–}5\) large data, 5–20 small). Loss for one target with positive \(c_{pos}\) and negatives \(c_{neg_1\dots k}\):

\[ L_{CE}=-\Big[\log\sigma(\mathbf c_{pos}\!\cdot\!\mathbf w)+\sum_{i=1}^{k}\log\sigma(-\mathbf c_{neg_i}\!\cdot\!\mathbf w)\Big] \]

Goal: maximise similarity of the true (target, context) pair and minimise similarity to noise words. Train by stochastic gradient descent.

Two embedding matrices & the gradients

SGNS learns a target matrix \(W\) and a context matrix \(C\); the final vector for word \(i\) is often \(\mathbf w_i+\mathbf c_i\). The SGD updates use the "prediction error" \((\sigma-t)\), where \(t=1\) for the positive word and \(0\) for negatives:

\[ \frac{\partial L}{\partial \mathbf w}=\sum_{j}\big(\sigma(\mathbf c_j\!\cdot\!\mathbf w)-t_j\big)\,\mathbf c_j \] \[ \mathbf w^{\text{new}}=\mathbf w-\eta\,\frac{\partial L}{\partial \mathbf w} \]

CBOW vs Skip-gram

🎯
Skip-gramPredicts context words from the centre word. Good with little data; represents rare words well.
📦
CBOWPredicts the centre word from surrounding context. Faster to train; slightly better on frequent words.

GloVe — global vectors

GloVe (Stanford) is unsupervised and uses the global word–word co-occurrence matrix \(X\), where \(X_{ij}\) = times word \(j\) occurs in the context of word \(i\). Its key idea: learn from ratios of co-occurrence probabilities \(P_{ik}/P_{jk}\), which encode meaning, rather than the raw probabilities. Unlike word2vec it captures global statistics, not just local context.

The parallelogram model: king − man + woman ≈ queen.

Window size mattersSmall windows (±2) → syntactically similar neighbours (Hogwarts → other schools). Large windows (±5) → topically related words (Hogwarts → Dumbledore, Malfoy).

Numerical — the SGNS update (exam favourite)

Corpus: "Ned Stark is the most honourable man". Positive pair (target, context) = (ned, stark); negative samples = pimples, zebra, idiot. Learning rate \(\eta=0.05\).

Target (input) embedding of ned: \(\mathbf w=[-0.018,\;0.404,\;-0.317]\).

Step 1 — Forward pass (dot product → sigmoid)

e.g. for stark: \(\mathbf c\cdot\mathbf w=0.116(-0.018)+0.723(0.404)+(-0.689)(-0.317)=0.508\), then \(\sigma(0.508)=0.624\).

Step 2 — Gradient w.r.t. the target embedding

\(\dfrac{\partial L}{\partial \mathbf w}=\sum_j(\sigma_j-t_j)\,\mathbf c_j\) — multiply each error by that word's context vector and add up:

  1. stark: \(-0.376\times[0.116,0.723,-0.689]=[-0.0436,-0.2718,0.2591]\)
  2. pimples: \(0.553\times[-0.94,0.601,0.146]=[-0.5198,0.3324,0.0807]\)
  3. zebra: \(0.534\times[-0.622,0.811,0.64]=[-0.3322,0.4331,0.3418]\)
  4. idiot: \(0.467\times[-0.077,-0.375,-0.056]=[-0.0360,-0.1751,-0.0262]\)
  5. Sum (column-wise): \([-0.9315,\;0.3185,\;0.6554]\)

Step 3 — Update the target embedding

\(\mathbf w^{\text{new}}=\mathbf w-\eta\,\dfrac{\partial L}{\partial \mathbf w}\), with \(\eta=0.05\):

  1. \(\eta\times\text{grad}=0.05\times[-0.9315,0.3185,0.6554]=[-0.0466,0.0159,0.0328]\)
  2. \(\mathbf w^{\text{new}}=[-0.018,0.404,-0.317]-[-0.0466,0.0159,0.0328]\)
Updated embedding of "ned": \([0.029,\;0.388,\;-0.35]\). Notice it moved toward stark's direction (positive) and away from the noise words — exactly what training should do.
🎯
Method to memorise (5 marks)① dot → sigmoid, ② error = σ − t (t=1 positive, 0 negative), ③ gradient = Σ(error)·(context vector), ④ new = old − η·gradient. Context embeddings update symmetrically with \(\partial L/\partial \mathbf c_j=(\sigma_j-t_j)\mathbf w\).
🌍

Real-life examples

Semantic search & recommendation — retrieve by meaning, not exact words.
Analogies — "Paris : France :: Tokyo : ?" → Japan, via vector arithmetic.
Historical semantics — train on each decade to watch word meanings drift (1850–1990 Google Books).
Bias auditing ⚠️ — embeddings learn cultural bias ("man : programmer :: woman : homemaker"). Using them in hiring tools can propagate that bias — an ethics point worth raising.
