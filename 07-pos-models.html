Statistical, ML & Neural Models of POS Tagging

How to actually decode the best tag sequence — Hidden Markov Models with the Viterbi algorithm, then discriminative MEMM/CRF, and finally neural Bi-LSTM-CRF, Transformers and LLM agents.

📘

Theory

The Hidden Markov Model (HMM)

A generative sequence model. States = tags \(t_1\dots t_N\); observations = words \(w_1\dots w_N\). It has three pieces:

Transition matrix \(A\): \(a_{ij}=P(t_i=j\mid t_{i-1}=i)\) — tag-to-tag.
Emission matrix \(B\): \(b_i(k)=P(w=v_k\mid t=i)\) — word given tag.
Initial vector \(\pi\): \(\pi_i=P(t_1=i)\).

Two assumptions make it tractable: (1) a tag depends only on the previous tag (bigram/Markov), (2) a word depends only on its own tag.

Why Bayes? Turning the problem around

We want \(\hat t_1^n=\arg\max_t P(t\mid w)\). Applying Bayes' rule and dropping the constant \(P(w)\):

\[ \hat t_1^{\,n}=\operatorname*{arg\,max}_{t}\underbrace{P(w\mid t)}_{\text{likelihood}}\,\underbrace{P(t)}_{\text{prior}}=\arg\max_t \prod_{i=1}^{n}\underbrace{P(w_i\mid t_i)}_{\text{emission}}\,\underbrace{P(t_i\mid t_{i-1})}_{\text{transition}} \]

So the joint score of a path is a product of emission × transition terms — the parameters you counted in Topic 6.

The Viterbi algorithm (dynamic programming)

Trying all \(|tags|^n\) sequences is exponential. Viterbi keeps, for each cell (state \(s\), time \(t\)), only the best path reaching it:

Initialise (column 1)
\[ v_1(s)=\pi_s\,b_s(w_1) \]
Recurse (column j ≥ 2)
\[ v_j(s)=\max_{s'}\;v_{j-1}(s')\,a_{s' s}\,b_s(w_j) \]

Store a backpointer to the \(s'\) that achieved each max; at the end, follow backpointers from the best final cell to read off the tag sequence.

MEMM — going discriminative

An HMM models the likelihood \(P(w\mid t)\). A Maximum Entropy Markov Model (MEMM) instead models the posterior \(P(t\mid w)\) directly with logistic regression, which lets you throw in rich features (word shape, suffixes, neighbouring words, previous tags). Decoding still uses Viterbi; the naive left-to-right greedy decoder is fast but can't use future evidence, so it underperforms.

Neural & modern approaches

Bi-LSTM-CRF — a Bi-LSTM reads the sentence both directions (word + char embeddings) to extract features; a CRF layer enforces valid tag sequences. ~97–98% on Penn Treebank.
Transformers (BERT/RoBERTa) — pretrained, then fine-tuned with a classification head. Self-attention captures deep context; ~98.5–99.5% accuracy, the current SOTA.
LLM prompting & agents — frame tagging as an instruction ("Tag using the Penn Treebank tagset…"), zero-/few-shot. In agents, a fast fine-tuned tagger is used as a tool to ground actions.
💡

Intuition

Think of the HMM as a storyteller: it secretly walks through tags (transitions) and at each tag "emits" a word (emission). Tagging reverses the story — given the words, find the most likely hidden walk. Viterbi does this efficiently by never keeping a path that's already worse than another path into the same state.

States emit words; transitions move between tags. Decode = find the best hidden state path.

Problem A — Markov chain (warm-up)

Weather states with \(P(\text{warm})_{\text{start}}=0.2\) and \(a_{\text{warm,warm}}=0.6\). Probability of four warm days in a row (state path 3-3-3-3):

  1. \(P=\pi_3\,(a_{33})^3=0.2\times(0.6)^3\)
  2. \(=0.2\times0.216\)
Answer: \(P(\text{warm}^4)=\mathbf{0.0432}\).

Problem B — "race": choosing VB vs NN

For "…to race tomorrow", compare the two tag hypotheses with NLTK-style probabilities:

  1. Verb path: \(P(\text{VB}\mid\text{TO})\,P(\text{NR}\mid\text{VB})\,P(\text{race}\mid\text{VB})=0.83\times0.0027\times0.00012=2.7\times10^{-7}\)
  2. Noun path: \(P(\text{NN}\mid\text{TO})\,P(\text{NR}\mid\text{NN})\,P(\text{race}\mid\text{NN})=0.00047\times0.0012\times0.00057=3.2\times10^{-10}\)
Answer: the verb path is ~840× larger, so we (correctly) tag race/VB.

⚠️
Viterbi pitfallsEach cell = (transition into it) × (emission of the current word) × (best previous cell). Take the max over previous states (not the sum — that's the forward algorithm), and remember to keep backpointers or you can't recover the path.
