Neural Language Models, LLMs & Prompt Engineering

From a single neuron to feedforward neural language models that beat N-grams through embeddings — then up to large language models, prompting, and transfer learning.

📘

Theory

The neural unit

A unit takes a weighted sum of inputs plus a bias, then applies a non-linear activation \(f\):

\[ z=\Big(\sum_i w_i x_i\Big)+b=\mathbf{w}\cdot\mathbf{x}+b,\qquad a=f(z) \]

Common activations: sigmoid \(\sigma(z)=\frac{1}{1+e^{-z}}\), tanh, and the most common, ReLU \(=\max(0,z)\).

The XOR problem → why we need hidden layers

A single perceptron is a linear classifier: \(w_1x_1+w_2x_2+b=0\) is just a line. AND and OR are linearly separable, but XOR is not — no single line separates it (Minsky & Papert, 1969). The fix: stack units into a layered network, whose hidden layer learns a new representation \(h\) in which the classes become separable.

Feedforward network & softmax

A 2-layer net (a.k.a. multi-layer perceptron): hidden layer \(\mathbf h=g(W\mathbf x+\mathbf b)\), output \(\mathbf y=\text{softmax}(U\mathbf h)\). Softmax generalises sigmoid to \(k\) classes:

\[ \text{softmax}(z_i)=\frac{e^{z_i}}{\sum_{j=1}^{k} e^{z_j}} \]

Neural language model

Task: predict word \(w_t\) from prior words \(w_{t-1},w_{t-2},w_{t-3},\dots\) Because sequences vary in length, use a fixed sliding window. Each context word becomes a one-hot vector, looked up in an embedding matrix \(E\); the embeddings are concatenated and fed forward:

\[ \mathbf e=[E\mathbf x_{1};\,E\mathbf x_{2};\,E\mathbf x_{3}] \] \[ \mathbf h=g(W\mathbf e+\mathbf b),\qquad \mathbf y=\text{softmax}(U\mathbf h) \]

You can either freeze \(E\) at pretrained word2vec values, or learn the embeddings jointly with the network.

One-hot inputs → embedding lookup → concatenate → hidden layer → softmax over the vocabulary.

From neural LM to LLM

State-of-the-art LMs replace the feedforward window with Transformers, trained on enormous corpora (pretraining). Transfer learning reuses a pretrained model on a new task; fine-tuning updates its weights on a smaller task-specific dataset.

💡

Intuition

Why neural LMs beat N-grams: they represent context words by embeddings, not raw identity, so similar words share strength.

🐱
The cat/dog exampleTraining saw "the cat gets fed" but never "dog gets fed". Test: "I forgot to feed the dog, so the dog gets ___". An N-gram can't predict fed (never seen "dog gets fed"). A neural LM knows cat ≈ dog in embedding space, so it generalises and predicts fed.

Trade-off: neural LMs handle longer histories and generalise better, but are more complex, slower, more energy-hungry, and less interpretable — so for small tasks an N-gram can still be the right tool.

Numerical / Problem

Problem A — Forward pass of one neuron

A unit has weights \(\mathbf w=[0.2,0.3,0.9]\), bias \(b=0.5\), input \(\mathbf x=[0.5,0.6,0.1]\). Find its sigmoid output.

  1. Weighted sum: \(\mathbf w\cdot\mathbf x=0.2(0.5)+0.3(0.6)+0.9(0.1)=0.10+0.18+0.09=0.37\).
  2. Add bias: \(z=0.37+0.5=0.87\).
  3. Sigmoid: \(a=\sigma(0.87)=\dfrac{1}{1+e^{-0.87}}=\dfrac{1}{1+0.419}\).
Answer: \(a\approx \mathbf{0.705}\).

Problem B — Softmax of an output vector

Given pre-activation scores \(\mathbf z=[1,\,2,\,3]\), compute the softmax distribution.

  1. Exponentiate: \(e^{1}=2.718,\; e^{2}=7.389,\; e^{3}=20.086\).
  2. Normaliser: sum \(=30.193\).
  3. Divide: \([2.718,\,7.389,\,20.086]/30.193\).
Answer: \([0.090,\,0.245,\,0.665]\) — sums to 1, and the largest score keeps the largest probability.

Real-life: LLMs & Prompt Engineering

Prompting reframes a task as a natural-language instruction to an LLM — often replacing fine-tuning entirely.

Zero-shot inference — give only the instruction, no examples: "Classify the sentiment: 'The movie dragged.' →"
Few-shot / in-context learning — include a few solved examples in the prompt, then the new input. The model learns the pattern from context, with no weight updates.
Instruction design — be explicit about format: "Output only the word/tag pairs."
🤖 ChatGPT / conversational agents ⌨️ Predictive text 🧑‍💻 Code assistants (CodeAid) 🧰 Generative AI assistants
🔁
Transfer learning vs fine-tuningTransfer learning freezes the pretrained weights and trains only new layers (e.g. reuse "recognise cars" knowledge to "recognise trucks"). Fine-tuning goes further and updates the pretrained layers too — useful when the new dataset is large and similar.
📌
One-line summaryNeuron = \(f(\mathbf w\!\cdot\!\mathbf x+b)\); stack units to solve non-linear problems like XOR; feed concatenated embeddings to predict the next word; scale up to Transformers/LLMs and steer them with zero-/few-shot prompts.
